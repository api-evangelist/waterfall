# Waterfall (waterfall)

Waterfall is a sequential software development methodology where progress flows steadily downward through defined phases including requirements, system design, implementation, testing, deployment, and maintenance. It emphasizes thorough documentation, fixed-scope planning, and phase-gate reviews before proceeding to subsequent stages. Waterfall remains widely used in regulated industries, government contracting, and projects with well-understood, stable requirements.

## Features

| Name | Description |
|------|-------------|
| Sequential Phase Execution | Each phase must be completed before the next begins, ensuring full documentation of outputs before proceeding. |
| Formal Phase-Gate Reviews | Structured review checkpoints allow stakeholders to approve progression before the next stage. |
| Comprehensive Documentation | Detailed documentation produced at each phase serves as the source of truth for subsequent phases. |
| Fixed Scope and Requirements | Requirements are gathered and locked upfront, providing high predictability for cost and schedule estimates. |
| Clear Milestones and Deliverables | Well-defined milestones provide clear checkpoints for project tracking and regulatory compliance. |

## Use Cases

| Name | Description |
|------|-------------|
| Government and Defense Contracts | Regulated government and defense projects with fixed requirements and strict documentation mandates. |
| Infrastructure and Construction Projects | Large systems with well-understood requirements and high change-management costs. |
| Compliance-Driven Software | Healthcare, finance, and safety-critical systems requiring audit trails and regulatory approval. |
| Fixed-Bid Contracts | Projects under fixed-price, fixed-scope contracts requiring up-front requirements definition. |
| System Integration Projects | Enterprise system integrations where interface contracts must be defined before implementation. |
